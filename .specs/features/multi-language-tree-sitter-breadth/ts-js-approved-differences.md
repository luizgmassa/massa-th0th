# TS/JS ETL Approved Differences

This ledger is executable: `structural-etl.test.ts` asserts every `AD-*` entry
used by the production adapter and fails if an undocumented difference appears.
Semantic chunking is not an approved difference and remains byte-for-byte frozen.

| ID | Previous regex projection | Structural projection | Reason |
| --- | --- | --- | --- |
| AD-001 | Calls inside methods used the enclosing class as caller. | Calls use the tightest containing method/function FQN. | Native spans make the actual lexical owner deterministic. |
| AD-002 | Definitions used only `file#name`. | Unique top-level definitions retain that form; nested and overloaded definitions use the versioned modern FQN. | Prevents nested/overload identity loss. |
| AD-003 | Symbol and reference locations were approximate lines. | Symbols and references retain exact UTF-8 byte/point spans and derive exact inclusive legacy lines. | Tree-sitter owns source coordinates. |
| AD-004 | Regex covered six declaration kinds and missed nested syntax. | The canonical structural taxonomy and nested declarations are projected. | Required normalized breadth replaces regex omissions. |
| AD-005 | Regex call matching admitted syntax-shaped false positives and guessed targets. | Query captures exclude declarations/control flow; unresolved and ambiguous targets remain explicit. | Resolution must never pick the first homonym or invent an FQN. |
| AD-006 | Imports lost aliases/forms and were also inferred as references. | Normalized bindings/forms drive one file dependency plus one compatibility reference per binding; query import edges are not duplicated. | Preserves dependency semantics without duplicate rows. |
| AD-007 | Parse/extractor failures became an empty successful file. | Native, query, ABI, and infrastructure failures reject the ETL run before Resolve/Load. Recovered trees retain structure, chunks, and diagnostics. | Empty graph success is unsafe. |
| AD-008 | Fingerprint-skipped files participated only through first-name-wins lookup. | Skipped files are never parsed; validated persisted TS/JS identities are generation-session candidates and ambiguity is retained. | Incremental resolution must not lose unchanged definitions. |
| AD-009 | Default exports and barrel forwarding were guessed from text. | Current-generation syntax metadata resolves them; legacy persisted rows lacking that metadata remain unresolved. | T9 must not fabricate missing persisted metadata. |
